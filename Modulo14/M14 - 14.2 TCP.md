# 14.2 — Descripcion General de TCP

---

## 🎯 Las 4 caracteristicas de TCP

> [!success] 1️⃣ Establece una sesion
> Protocolo **orientado a la conexion**. Negocia y establece una conexion antes de enviar cualquier trafico (3-way handshake).

> [!success] 2️⃣ Garantiza entrega confiable
> Si un segmento se pierde o corrompe, TCP asegura que **llegue de todas formas** (retransmision).

> [!success] 3️⃣ Entrega en el mismo orden
> Las redes pueden tener rutas con velocidades distintas — los datos pueden llegar desordenados. TCP los **reordena** usando numeros de secuencia.

> [!success] 4️⃣ Control de flujo
> Si el receptor esta sobrecargado, TCP le pide al emisor que **reduzca la velocidad**.

---

## 🔬 TCP es "con estado" (stateful)

> [!info] Que significa "con estado"
> TCP **rastrea el estado** de la sesion — que se envio, que se confirmo, que falta. No es un protocolo que "dispara y olvida".

## 📦 Encabezado TCP — Los campos

```
  ┌──────────────────┬──────────────────┐
  │  Puerto Origen    │  Puerto Destino   │  16b cada uno
  ├──────────────────┴──────────────────┤
  │       Numero de Secuencia            │  32 bits
  ├───────────────────────────────────────┤
  │       Numero de Confirmacion (ACK)   │  32 bits
  ├────┬────┬──────────┬─────────────────┤
  │Long│Res.│Bits Ctrl │  Tamaño Ventana  │
  ├────┴────┴──────────┴─────────────────┤
  │  Checksum          │  Urgente         │
  └──────────────────────────────────────┘
```

| Campo | Bits | Funcion |
|-------|------|---------|
| **Puerto origen** | 16 | Identifica la app de origen |
| **Puerto destino** | 16 | Identifica la app de destino |
| **Numero de secuencia** | 32 | Reensamblar datos en orden |
| **Numero de confirmacion** | 32 | Siguiente byte esperado |
| **Longitud del encabezado** | 4 | Tamaño del header |
| **Bits de control** | 16 | Flags: SYN, ACK, FIN... |
| **Tamaño de ventana** | 16 | Bytes que se pueden aceptar |
| **Checksum** | 16 | Deteccion de errores |
| **Urgente** | 16 | Indica si los datos son urgentes |

> [!tip] 💡 Por que TCP tiene overhead
> Comparado con UDP, TCP tiene un header mucho mas grande (20+ bytes vs 8 bytes de UDP) porque incluye toda la maquinaria de confiabilidad: secuencia, ACK, ventana, control.
