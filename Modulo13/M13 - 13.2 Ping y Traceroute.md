# 13.2 — Pruebas de Ping y Traceroute

---

## 🏓 Ping — Prueba de conectividad

> [!info] Como funciona
> Usa **Echo Request** y **Echo Reply** de ICMP para probar conectividad. Da un resumen con tasa de exito y tiempo de ida y vuelta (RTT).

> [!tip] 💡 El primer ping frecuentemente falla
> Si se necesita resolucion de direcciones (ARP en IPv4 o ND en IPv6) antes del Echo Request, el primer intento puede agotar el tiempo de espera. Es normal.

---

## 🎯 Las 3 pruebas progresivas de ping

> [!success] 1️⃣ Ping al Loopback — prueba la PC misma
> ```
> ping 127.0.0.1    (IPv4)
> ping ::1          (IPv6)
> ```
> - Respuesta → IP esta correctamente instalado en el host
> - Error → TCP/IP no funciona en el host (problema de driver/stack)

> [!success] 2️⃣ Ping al Default Gateway — prueba la red local
> ```
> ping [IP del gateway]
> ```
> - Exitoso → el host y el router estan operativos en la LAN
> - Si no responde → probar ping a otro host conocido de la LAN

> [!success] 3️⃣ Ping a un host remoto — prueba la interconexion completa
> ```
> ping [IP de host en otra red]
> ```
> - Exitoso → confirma comunicacion a traves de toda la ruta

> [!warning] ⚠️ Sin respuesta no siempre significa "caido"
> Muchos administradores **bloquean ICMP** por seguridad. Un ping fallido puede ser una restriccion, no una falla real.

---

## 🩺 La escalera de diagnostico con ping

```
  PASO 1  ping 127.0.0.1        → ¿funciona el stack TCP/IP local?
       ↓ OK
  PASO 2  ping [Default Gateway] → ¿funciona la LAN?
       ↓ OK
  PASO 3  ping [host remoto]     → ¿funciona la ruta completa?
```

---

## 🗺 Traceroute — Prueba la ruta completa

> [!info] Que hace traceroute
> Muestra la lista de **saltos** (routers) atravesados hasta el destino, con el tiempo de ida y vuelta de cada uno.

> [!warning] ⚠️ El asterisco (*)
> Indica un paquete perdido o sin respuesta en ese salto. Ayuda a localizar el router problematico en la ruta.

---

## ⚙️ Como funciona traceroute — el truco del TTL

> [!example] Incremento progresivo del TTL
> ```
> Mensaje 1: TTL=1 → expira en Router1 → responde "Tiempo excedido"
>            → conocemos el salto 1
>
> Mensaje 2: TTL=2 → expira en Router2 → responde "Tiempo excedido"
>            → conocemos el salto 2
>
> Mensaje 3: TTL=3 → expira en Router3 → responde "Tiempo excedido"
>            → conocemos el salto 3
>
> ... continua incrementando hasta llegar al destino
>     o alcanzar el maximo predefinido
> ```

> [!tip] 💡 IPv4 usa TTL, IPv6 usa Hop Limit
> El mismo principio, campo diferente segun la version de IP.

```cisco
! Windows
tracert 8.8.8.8

! Cisco IOS / Linux / Mac
traceroute 8.8.8.8
```
