# 13.1 — Mensajes ICMP

---

## 📬 Que es ICMP

> [!info] Internet Control Message Protocol
> Proporciona informacion sobre **problemas** relacionados con el procesamiento de paquetes IP. No transporta datos de usuario — solo avisa cuando algo sale mal.

```
╔═══════════════════════╦═══════════════════════════════════╗
║  ICMPv4               ║  ICMPv6                           ║
║  Mensajeria para IPv4 ║  Mensajeria para IPv6              ║
║                       ║  + funcionalidad adicional (ND)   ║
╚═══════════════════════╩═══════════════════════════════════╝
```

> [!warning] ⚠️ ICMPv4 no es obligatorio
> Muchas redes **bloquean ICMP** por razones de seguridad — un ping sin respuesta no siempre significa que el host este caido.

---

## 🎯 Los 3 mensajes comunes a ICMPv4 e ICMPv6

> [!success] 1️⃣ Accesibilidad al host (Echo)
> Prueba si un host esta disponible.
> ```
> Host local ──Echo Request──► Host destino
> Host local ◄──Echo Reply──── Host destino
> ```
> Esta es la base del comando **ping**.

> [!danger] 2️⃣ Destino o servicio inalcanzable
> Notifica al origen que el destino/servicio no es accesible. Incluye un **codigo** que explica por que.

| Codigo ICMPv4 | Significado |
|---------------|-------------|
| 0 | Red inalcanzable |
| 1 | Host inalcanzable |
| 2 | Protocolo inalcanzable |
| 3 | Puerto inalcanzable |

| Codigo ICMPv6 | Significado |
|---------------|-------------|
| 0 | No hay ruta al destino |
| 1 | Comunicacion prohibida (firewall) |
| 2 | Mas alla del alcance de la IP origen |
| 3 | No se puede alcanzar la direccion |
| 4 | Puerto inalcanzable |

> [!warning] 3️⃣ Tiempo excedido
> Cuando el **TTL** (IPv4) o **Hop Limit** (IPv6) llega a 0, el router envia este mensaje al origen.
> Es la base de la herramienta **traceroute**.

---

## 🆕 Mensajes exclusivos de ICMPv6 — Neighbor Discovery (ND)

> [!info] 4 nuevos protocolos dentro de ND
> ICMPv6 agrega funciones que ICMPv4 no tiene, agrupadas en el protocolo **Neighbor Discovery**.

> [!example] Entre router y dispositivo (asignacion de direcciones)
> - **RS** (Router Solicitation) → el host pregunta si hay routers
> - **RA** (Router Advertisement) → el router responde con info de red (cada ~200 segundos)

> [!example] Entre dispositivos (deteccion y resolucion)
> - **NS** (Neighbor Solicitation) → pregunta por una MAC o verifica unicidad
> - **NA** (Neighbor Advertisement) → responde con la MAC o confirma duplicado

---

## 🔄 Flujo RS/RA completo

> [!example] PC1 arranca y necesita configurarse
> ```
> PC1 → RS: "Hola, acabo de arrancar. ¿Hay un router IPv6?
>            Necesito la info de mi direccion."
>
> R1  → RA: "Hola, soy R1. Usa SLAAC con el prefijo
>            2001:db8:acad:1::/64. Mi LLA fe80::1
>            es tu gateway predeterminado."
> ```

---

## 🔍 DAD — Deteccion de Direcciones Duplicadas

> [!info] Verificar que una IPv6 sea unica
> El dispositivo envia un **NS** con su propia IPv6 como direccion objetivo.
> - Si nadie responde → la direccion es unica ✅
> - Si otro dispositivo responde con **NA** → hay un duplicado ⚠️

> [!note] No es obligatorio pero se recomienda (RFC 4861)

---

## 🔍 Resolucion de MAC via NS/NA (equivalente a ARP)

> [!example] R1 necesita la MAC de 2001:db8:acad:1::10
> ```
> R1 → NS a direccion de nodo solicitado: "¿Cual es tu MAC?"
> Dispositivo 2001:db8:acad:1::10 → NA: "Mi MAC es XX:XX..."
> ```
