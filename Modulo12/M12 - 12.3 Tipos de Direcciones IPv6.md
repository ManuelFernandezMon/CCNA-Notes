# 12.3 — Tipos de Direcciones IPv6

---

## 📂 Las 3 categorias

> [!info] 1️⃣ Unicast
> Identifica de forma unica **una interfaz** de un dispositivo IPv6.

> [!info] 2️⃣ Multicast
> Envia un paquete a **multiples destinos** (grupo suscrito).

> [!info] 3️⃣ Anycast
> Direccion unicast asignada a **varios dispositivos**. El paquete va al **mas cercano**.

> [!warning] ⚠️ IPv6 NO tiene broadcast
> En su lugar usa la direccion **multicast de todos los nodos** (FF02::1), que da el mismo resultado.

---

## 📏 Longitud de prefijo IPv6

> [!info] Notacion de barra (igual que IPv4)
> Indica la porcion de red. Va de 0 a 128.
> **Recomendado: /64** para LANs y la mayoria de redes.

> [!tip] 💡 Por que /64
> SLAAC usa 64 bits para el ID de interfaz. Usar /64 facilita la autoconfiguracion y la administracion de subredes.

---

## 🌍 Las 2 direcciones Unicast de cada dispositivo

> [!success] 🌐 GUA — Global Unicast Address
> - Como una **IP publica IPv4** — enrutable en Internet
> - Globalmente unica
> - Empieza con **2000::/3** (decimal 2 o 3)

> [!info] 🔗 LLA — Link-Local Address
> - Solo para comunicarse en el **mismo enlace local** (subred)
> - **NO enrutable** — confinada a un solo enlace
> - **OBLIGATORIA** en toda interfaz IPv6
> - Empieza con **FE80::/10**
> - Si no se configura, el dispositivo la crea automaticamente

```
  Un dispositivo IPv6 tipico:
  GUA: 2001:db8:acad:1::100/64   ← para Internet
  LLA: fe80::1                    ← para el enlace local
```

---

## 🏗 Estructura de una GUA

```
  2001:db8:acad : 0001 : 0000:0000:0000:0100
  └──────┬──────┘ └─┬─┘  └────────┬─────────┘
  Prefijo de       ID de         ID de Interfaz
  enrutamiento     Subred        (64 bits, = porcion host)
  global (asignado (la org crea
  por el ISP)      sus subredes)
```

| Campo | Funcion |
|-------|---------|
| **Prefijo global** | Asignado por el ISP (la "red") |
| **ID de subred** | La organizacion crea sus subredes |
| **ID de interfaz** | Equivale a la porcion host de IPv4 (/64) |

---

## 🔒 Nota sobre ULA (Local Unica)

> [!note] FC00::/7 — similar a las privadas RFC 1918
> Las ULA se usan para direccionamiento local dentro de un sitio. No se enrutan globalmente. Pero NO fueron diseñadas para "ocultar" la red por seguridad (eso es un mal uso comun).
