# 12.1 — Problemas de IPv4

---

## 🚨 Por que necesitamos IPv6

> [!danger] IPv4 se quedo sin direcciones
> IPv4 tiene un maximo teorico de **4.3 mil millones** de direcciones. Con la poblacion creciente de Internet, NAT como parche, y el Internet de las Cosas (IoT), llego el momento de migrar.

> [!success] IPv6 es el sucesor
> - Espacio de **128 bits** (vs 32 de IPv4)
> - Correcciones a las limitaciones de IPv4
> - Mejoras de diseño (sin NAT, autoconfiguracion, seguridad integrada)

---

## 🔄 Coexistencia IPv4 + IPv6

> [!info] La transicion lleva años
> Ambos protocolos coexistiran. El IETF creo 3 tecnicas de migracion:

> [!example] 1️⃣ Dual Stack (pila doble)
> El dispositivo ejecuta **IPv4 e IPv6 simultaneamente**.
> ```
> Dispositivo ──► IPv4 stack ──► red IPv4
>            └──► IPv6 stack ──► red IPv6
> ```
> Es el metodo preferido.

> [!example] 2️⃣ Tunneling (tunelizacion)
> Transporta un paquete IPv6 **encapsulado dentro** de un paquete IPv4.
> ```
> [IPv4 header | [IPv6 packet] ]
>   atraviesa una red IPv4
> ```

> [!example] 3️⃣ Translation (traduccion)
> **NAT64** permite que dispositivos IPv6 se comuniquen con dispositivos IPv4 (similar a NAT en IPv4).

> [!tip] 💡 El objetivo final
> Tunneling y translation son solo para la transicion. La meta es **IPv6 nativo** de origen a destino.
