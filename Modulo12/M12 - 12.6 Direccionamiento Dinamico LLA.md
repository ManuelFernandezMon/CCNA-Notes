# 12.6 — Direccionamiento Dinamico para LLA IPv6

---

## 🔗 LLAs dinamicas

> [!info] Toda interfaz IPv6 debe tener una LLA
> Si no se configura manualmente, se crea **automaticamente**:
> ```
> Prefijo FE80::/10  +  ID de interfaz (EUI-64 o aleatorio)
> = LLA dinamica
> ```

---

## 🖥 LLAs dinamicas en Windows

> [!example] EUI-64 vs aleatorio en ipconfig
> ```
> Con EUI-64 (nota el ff:fe en el medio):
> IPv6:       2001:db8:acad:1:fc99:47ff:fe75:cee0
> Link-local: fe80::fc99:47ff:fe75:cee0
> Gateway:    fe80::1
> ```
> ```
> Con ID aleatorio:
> IPv6:       2001:db8:acad:1:50a5:8a35:a5bb:66e1
> Link-local: fe80::50a5:8a35:a5bb:66e1
> Gateway:    fe80::1
> ```

> [!tip] 💡 Nota el gateway
> El gateway predeterminado es la **LLA del router** (fe80::1) — no su GUA.

---

## 🔧 LLAs dinamicas en routers Cisco

> [!info] Automatico al asignar una GUA
> Los routers Cisco crean una LLA **automaticamente** cuando se asigna una GUA a la interfaz. Por defecto usan **EUI-64**.

```cisco
R1# show ipv6 interface brief

GigabitEthernet0/0/0 [up/up]
    FE80::7279:B3FF:FE92:3640   ← LLA (automatica, EUI-64)
    2001:DB8:ACAD:1::1          ← GUA (configurada)
```

> [!note] Verificacion
> El comando `show ipv6 interface brief` muestra ambas direcciones: la LLA (FE80::) y la GUA (2001::).
