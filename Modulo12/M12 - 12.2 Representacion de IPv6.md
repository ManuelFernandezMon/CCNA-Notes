# 12.2 — Representacion de Direcciones IPv6

---

## 🔤 Formato de una direccion IPv6

> [!info] 128 bits en hexadecimal
> - Escrita en hexadecimal (no distingue mayusculas/minusculas)
> - Formato: `x:x:x:x:x:x:x:x` donde cada `x` = 4 valores hex
> - Un **hexteto** = segmento de 16 bits (4 digitos hex)

```
  2001:0db8:0000:1111:0000:0000:0000:0200
  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘ └──┘
   1    2    3    4    5    6    7    8
  8 hextetos × 16 bits = 128 bits
```

---

## ✂️ Regla 1 — Omitir ceros iniciales

> [!success] Quita los 0 al INICIO de cada hexteto
> ```
> 01ab → 1ab
> 09f0 → 9f0
> 00ab → ab
> 0000 → 0
> ```

> [!danger] ⚠️ Solo ceros INICIALES, nunca finales
> Si quitaras ceros finales la direccion seria ambigua.

> [!example] Aplicando la Regla 1
> ```
> Original: 2001:0db8:0000:1111:0000:0000:0000:0200
> Regla 1:  2001:db8:0:1111:0:0:0:200
> ```

---

## ✂️ Regla 2 — Dos puntos dobles (::)

> [!success] :: reemplaza una cadena de ceros consecutivos
> El `::` puede reemplazar uno o mas hextetos contiguos compuestos **solo de ceros**.

> [!danger] ⚠️ Solo se puede usar UNA VEZ por direccion
> Si lo usaras dos veces, no se sabria cuantos ceros representa cada uno.

> [!example] Aplicando ambas reglas
> ```
> Original:    2001:0db8:0000:1111:0000:0000:0000:0200
> Regla 1:     2001:db8:0:1111:0:0:0:200
> Regla 2:     2001:db8:0:1111::200      ✅ formato final
> ```
> ```
> Otro ejemplo:
> 2001:db8:cafe:1:0:0:0:1  →  2001:db8:cafe:1::1
> ```
