# 12.4 — Configuracion Estatica de GUA y LLA

---

## 💡 Regla general de comandos IPv6

> [!tip] IPv6 = IPv4 cambiando "ip" por "ipv6"
> La mayoria de comandos IPv6 de Cisco son identicos a los de IPv4 — solo se usa `ipv6` en lugar de `ip`.

---

## 🌐 Configurar GUA en un router

```cisco
R1(config)# interface gigabitethernet 0/0/0
R1(config-if)# ipv6 address 2001:db8:acad:1::1/64
R1(config-if)# no shutdown
R1(config-if)# exit
```

> [!info] Estructura del comando
> `ipv6 address [direccion]/[longitud-prefijo]`

---

## 🖥 Configurar GUA en un host Windows

> [!info] Similar a configurar IPv4 manualmente
> Se introduce la direccion IPv6, longitud de prefijo y gateway.

> [!tip] 💡 Mejor practica para el gateway
> Usar la **LLA del router** como gateway predeterminado.
> Cuando se usa SLAAC o DHCPv6, la LLA del router se especifica automaticamente como gateway.

---

## 🔗 Configurar LLA manualmente

> [!info] Para crear una LLA reconocible
> Por defecto la LLA se genera sola, pero puedes configurarla manual para que sea facil de recordar.

```cisco
R1(config)# interface gigabitethernet 0/0/0
R1(config-if)# ipv6 address fe80::1:1 link-local
R1(config-if)# no shutdown
R1(config-if)# exit
```

> [!tip] 💡 Practica comun
> Configurar una LLA **diferente en cada interfaz** del router (fe80::1:1, fe80::2:1...) para identificar facilmente el router y la interfaz.
