# 12.8 — Subnetear una Red IPv6

---

## 🎨 IPv6 fue diseñado pensando en subredes

> [!success] Mucho mas simple que IPv4
> IPv6 tiene un campo **ID de subred dedicado** dentro de la GUA. No hay que tomar bits prestados ni calcular mascaras complicadas.

```
  2001:db8:acad : 0001 : interfaz...
  └─────┬──────┘ └──┬─┘
  Prefijo global   ID de Subred
  (del ISP)        (tu la incrementas)
```

---

## 🔢 Ejemplo de subneteo IPv6

> [!example] Prefijo 2001:db8:acad::/48 con ID de subred de 16 bits
> ```
> Prefijo global: 2001:db8:acad  (/48, fijo en todas las subredes)
> ID de subred:   16 bits → permite 65,536 subredes /64
>
> Solo incrementas el hexteto de la ID de subred:
> Subred 1: 2001:db8:acad:0001::/64
> Subred 2: 2001:db8:acad:0002::/64
> Subred 3: 2001:db8:acad:0003::/64
> ...
> ```

> [!tip] 💡 La belleza de IPv6
> Cada subred /64 da mas direcciones de las que jamas necesitaras. No hay que preocuparse por el desperdicio — hay direcciones de sobra.

---

## 🔧 Configurar subredes IPv6 en un router

> [!example] R1 con 3 subredes diferentes
> ```cisco
> R1(config)# interface gigabitethernet 0/0/0
> R1(config-if)# ipv6 address 2001:db8:acad:1::1/64
> R1(config-if)# no shutdown
> R1(config-if)# exit
>
> R1(config)# interface gigabitethernet 0/0/1
> R1(config-if)# ipv6 address 2001:db8:acad:2::1/64
> R1(config-if)# no shutdown
> R1(config-if)# exit
>
> R1(config)# interface serial 0/1/0
> R1(config-if)# ipv6 address 2001:db8:acad:3::1/64
> R1(config-if)# no shutdown
> ```
> Cada interfaz en una subred distinta (::1, ::2, ::3).

---

## ⚖️ IPv6 vs IPv4 subnetting

| | IPv4 | IPv6 |
|--|------|------|
| **Metodo** | Tomar bits prestados de host | Campo ID de subred dedicado |
| **Calculo** | Numero magico, mascaras | Solo incrementar el hexteto |
| **Desperdicio** | Preocupacion constante | Irrelevante (hay de sobra) |
| **Prefijo tipico** | Variable (/24, /26...) | Casi siempre /64 |
