# 12.5 — Direccionamiento Dinamico para GUA IPv6

---

## 📡 Mensajes RS y RA (ICMPv6)

> [!info] Como los dispositivos descubren como configurarse
> ```
> RS (Router Solicitation) → el host pregunta: "¿Hay routers aqui?"
> RA (Router Advertisement) → el router responde con info de red
> ```
> Los routers envian RA periodicamente (cada ~200 segundos).

> [!info] El RA proporciona
> - Prefijo de red y longitud del prefijo
> - Direccion del gateway predeterminado
> - Direcciones DNS y nombre de dominio

---

## 🔀 Los 3 metodos del mensaje RA

> [!success] Metodo 1 — SLAAC (puro)
> **Stateless Address Autoconfiguration**. El dispositivo se configura **sin servidor DHCPv6**.
> - El prefijo lo da el RA
> - El dispositivo crea su propio ID de interfaz (EUI-64 o aleatorio)

> [!info] Metodo 2 — SLAAC + DHCPv6 stateless
> - SLAAC crea la GUA
> - Un servidor DHCPv6 stateless da info adicional (DNS, nombre de dominio)

> [!info] Metodo 3 — DHCPv6 stateful
> Similar a DHCP de IPv4. El servidor DHCPv6 da **todo**: GUA, prefijo, DNS.

```
╔════════════════════╦═══════════════════════════════════════╗
║ METODO             ║ ¿Quien da la GUA?                    ║
╠════════════════════╬═══════════════════════════════════════╣
║ SLAAC              ║ El dispositivo mismo (sin DHCP)       ║
║ SLAAC + DHCPv6     ║ SLAAC la GUA, DHCP la info extra      ║
║ DHCPv6 stateful    ║ El servidor DHCPv6 (como IPv4)        ║
╚════════════════════╩═══════════════════════════════════════╝
```

## 🔧 EUI-64 — Generar el ID de interfaz desde la MAC

> [!info] El proceso EUI-64 del IEEE
> Convierte una MAC de 48 bits en un ID de interfaz de 64 bits:
> ```
> PASO 1: Inserta "FFFE" en el medio de la MAC
> PASO 2: Invierte el 7º bit (de 0 a 1)
> ```

> [!example] Ejemplo de EUI-64
> ```
> MAC original:  fc:99:47:75:ce:e0
> Inserta FFFE:  fc99:47ff:fe75:cee0
> Invierte 7bit: fe99:47ff:fe75:cee0   ← ID de interfaz
> ```

> [!tip] 💡 Como reconocer EUI-64
> Si ves **ff:fe** en el medio del ID de interfaz, se genero con EUI-64.

---

## 🎲 ID de interfaz aleatorio

> [!info] Alternativa a EUI-64
> Por privacidad, muchos SO generan un ID **aleatorio** en vez de usar la MAC.
> Desde **Windows Vista**, Windows usa ID aleatorio por defecto.

> [!note] 🔍 DAD — Deteccion de Direcciones Duplicadas
> Para garantizar que la direccion sea unica, el dispositivo usa DAD (similar a un ARP a su propia direccion). Si no hay respuesta, la direccion es unica.
