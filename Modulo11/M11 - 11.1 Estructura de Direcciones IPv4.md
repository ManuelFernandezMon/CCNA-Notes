# 11.1 — Estructura de Direcciones IPv4

---

## 🧩 Porcion de Red y Porcion de Host

> [!info] Una direccion jerarquica de 32 bits
> Toda IPv4 tiene dos partes: la **porcion de red** (que red) y la **porcion de host** (que dispositivo). La mascara de subred define donde termina una y empieza la otra.

```
  192  .  168  .   10  .   25
  └──────────────────┘   └────┘
     PORCION DE RED      PORCION HOST
   (que red es esta)   (que dispositivo)
```

---

## 🎭 La Mascara de Subred

> [!info] Define red vs host
> La mascara se compara con la IP **bit por bit, de izquierda a derecha**.
> - Bits en **1** = porcion de RED
> - Bits en **0** = porcion de HOST

```
  IP:       192.168.10.25
  Mascara:  255.255.255.0

  Binario:
  IP:       11000000.10101000.00001010.00011001
  Mascara:  11111111.11111111.11111111.00000000
            └────── RED (24 bits) ─────┘└HOST(8)┘
```

---

## 📏 La Longitud de Prefijo (notacion de barra)

> [!tip] Forma compacta de escribir la mascara
> El prefijo cuenta cuantos bits estan en **1** en la mascara.

| Mascara | Binario | Prefijo |
|---------|---------|---------|
| 255.0.0.0 | 11111111.0.0.0 | /8 |
| 255.255.0.0 | 11111111.11111111.0.0 | /16 |
| 255.255.255.0 | ...11111111.0 | /24 |
| 255.255.255.128 | ...10000000 | /25 |
| 255.255.255.192 | ...11000000 | /26 |
| 255.255.255.224 | ...11100000 | /27 |
| 255.255.255.252 | ...11111100 | /30 |

## 🔢 ANDing — Como se calcula la direccion de red

> [!warning] La operacion AND logica
> Solo `1 AND 1 = 1`. Cualquier otra combinacion = 0.
> ```
> 1 AND 1 = 1
> 1 AND 0 = 0
> 0 AND 1 = 0
> 0 AND 0 = 0
> ```

> [!example] Calcular la red de 192.168.10.25 /24
> ```
> IP:       11000000.10101000.00001010.00011001
> Mascara:  11111111.11111111.11111111.00000000
> ────────────────────────────────────── AND
> RED:      11000000.10101000.00001010.00000000
>         = 192.168.10.0   ← direccion de red ✅
> ```

---

## 🏠 Las 3 direcciones especiales de cada red

> [!info] En toda subred hay 3 direcciones con rol fijo
> Ejemplo en `192.168.10.0/24`:

```
╔═══════════════════════╦══════════════════╦═══════════════════╗
║ DIRECCION DE RED      ║ 192.168.10.0     ║ todos los bits    ║
║ (identifica la red)   ║                  ║ de host en 0      ║
╠═══════════════════════╬══════════════════╬═══════════════════╣
║ PRIMER HOST UTILIZABLE║ 192.168.10.1     ║ ...00000001       ║
║ ULTIMO HOST UTILIZABLE║ 192.168.10.254   ║ ...11111110       ║
╠═══════════════════════╬══════════════════╬═══════════════════╣
║ BROADCAST             ║ 192.168.10.255   ║ todos los bits    ║
║ (a todos en la red)   ║                  ║ de host en 1      ║
╚═══════════════════════╩══════════════════╩═══════════════════╝
```

> [!danger] ⚠️ Estas 2 direcciones NO se asignan a hosts
> La **direccion de red** (todos 0) y la **direccion de broadcast** (todos 1) estan reservadas. Por eso siempre restas 2 al calcular hosts utilizables.
