# 11.3 — Tipos de Direcciones IPv4

---

## 🌍 Publicas vs Privadas

> [!info] Publicas — enrutables en Internet
> Se enrutan globalmente entre routers de los ISPs. Unicas en todo el mundo. Asignadas por IANA/RIRs.

> [!success] Privadas — solo redes internas (RFC 1918)
> NO se enrutan en Internet. Cualquier red interna puede usarlas. No son unicas.

```
╔═══════════════════════╦═══════════════════════════════════════╗
║ Red y prefijo         ║ Rango privado (RFC 1918)             ║
╠═══════════════════════╬═══════════════════════════════════════╣
║ 10.0.0.0/8            ║ 10.0.0.0    →  10.255.255.255         ║
║ 172.16.0.0/12         ║ 172.16.0.0  →  172.31.255.255         ║
║ 192.168.0.0/16        ║ 192.168.0.0 →  192.168.255.255        ║
╚═══════════════════════╩═══════════════════════════════════════╝
```

---

## 🔄 NAT — Como las privadas llegan a Internet

> [!info] Network Address Translation
> NAT traduce las direcciones privadas internas a una direccion publica. Normalmente habilitado en el router perimetral que conecta a Internet.
> ```
> PC privada (192.168.1.10) ──► Router NAT ──► IP publica ──► 🌐
>                              traduce privada → publica
> ```

---

## 🔧 Direcciones reservadas especiales

> [!example] 🔁 Loopback — 127.0.0.0/8
> Comunmente `127.0.0.1`. Se usa para **probar si TCP/IP funciona** en el propio host.
> ```
> ping 127.0.0.1 → si responde, la pila TCP/IP funciona
> ```

> [!warning] 🏷 Link-local (APIPA) — 169.254.0.0/16
> Direcciones auto-asignadas. Windows las usa cuando **no hay servidor DHCP disponible**.
> ```
> ¿Ves 169.254.x.x en ipconfig? → DHCP fallo
> ```

---

## 📜 Direccionamiento con clases (legacy)

> [!note] RFC 790 (1981) — ya obsoleto
> Antes las IPs se asignaban en clases fijas:

| Clase | Rango | Uso |
|-------|-------|-----|
| **A** | 0.0.0.0 – 127.x | Redes enormes |
| **B** | 128.0 – 191.255.x | Redes medianas |
| **C** | 192.0.0 – 223.255.255.x | Redes pequeñas |
| **D** | 224.0.0 – 239.x | Multicast |
| **E** | 240.0.0 – 255.x | Experimental |

> [!danger] ⚠️ Por que se abandono
> El direccionamiento con clases **desperdiciaba muchas IPs**. Se reemplazo con direccionamiento **sin clases** (CIDR) que ignora las reglas de clases A/B/C.

---

## 🏛 Quien asigna las IPs

```
  IANA (autoridad global)
    │
    ├── AfriNIC  (Africa)
    ├── APNIC    (Asia-Pacifico)
    ├── ARIN     (America del Norte)
    ├── LACNIC   (America Latina)  ← Panama esta aqui
    └── RIPE NCC (Europa)
         │
         └── ISPs → organizaciones y usuarios finales
```
