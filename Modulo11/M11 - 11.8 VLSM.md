# 11.8 — VLSM (Mascara de Longitud Variable)

---

## 💡 El problema que resuelve VLSM

> [!warning] El subneteo tradicional desperdicia direcciones
> Con subneteo tradicional, **todas las subredes tienen la misma mascara**. Esto desperdicia direcciones en enlaces que necesitan pocas.

> [!example] El desperdicio en enlaces WAN
> ```
> Un enlace WAN punto a punto solo necesita 2 direcciones
> (una para cada router)
>
> Pero con mascara /27 (30 hosts):
> → desperdicia 28 direcciones por enlace
> → 3 enlaces WAN = 84 direcciones desperdiciadas
> ```

---

## ✂️ Que es VLSM

> [!success] Variable Length Subnet Mask
> VLSM permite **subnetear una subred** — usar diferentes mascaras para diferentes segmentos segun lo que cada uno necesita.

```
  SUBNETEO TRADICIONAL          VLSM
  ────────────────────          ────────────────────
  Todas las subredes /27        LANs grandes:  /27 (30 hosts)
  (mismo tamaño)                LANs medianas: /28 (14 hosts)
                                Enlaces WAN:   /30 (2 hosts)
  Desperdicia en WAN            Sin desperdicio ✅
```

---

## 🎯 La regla de oro de VLSM

> [!danger] ⚠️ SIEMPRE de mayor a menor
> Empieza satisfaciendo la subred **mas grande** primero, luego continua hacia las mas pequeñas.

```
  ORDEN CORRECTO:
  1º La LAN con mas hosts    → asigna primero
  2º La siguiente mas grande
  3º ...
  Ultimo: los enlaces WAN /30 (solo 2 hosts)
```

---

## 📐 Ejemplo completo de VLSM

> [!example] Topologia con 7 subredes (4 LANs + 3 WANs)
> ```
> Red base: 192.168.20.0/24
>
> Edificio D: 28 hosts → /27 (30 hosts) → 192.168.20.0/27
> Edificio C: 14 hosts → /28 (14 hosts) → 192.168.20.32/28
> Edificio B: 12 hosts → /28 (14 hosts) → 192.168.20.48/28
> Edificio A: 7 hosts  → /28 (14 hosts) → 192.168.20.64/28
>
> WAN 1: 2 hosts → /30 → 192.168.20.80/30
> WAN 2: 2 hosts → /30 → 192.168.20.84/30
> WAN 3: 2 hosts → /30 → 192.168.20.88/30
>
> Resultado: sin desperdicio, todo cabe en el /24 ✅
> ```

---

## 🔄 Comparativa visual

```
  TRADICIONAL (/27 para todo)      VLSM
  ──────────────────────────       ──────────────────────────
  LAN-A /27 → 30 hosts ✅          LAN-A /27 → 30 hosts ✅
  LAN-B /27 → 30 hosts ✅          LAN-B /28 → 14 hosts ✅
  WAN-1 /27 → 30 hosts ❌          WAN-1 /30 → 2 hosts ✅
  (28 desperdiciadas)              (0 desperdiciadas)
```

> [!tip] 💡 Cuando usar VLSM
> Siempre que tengas segmentos de **tamaños muy diferentes** — especialmente LANs grandes mezcladas con enlaces WAN punto a punto. Es el estandar en redes reales.
