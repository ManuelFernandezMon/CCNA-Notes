# 11.7 — Subnetear para Cumplir Requisitos

---

## 🏢 Privadas vs Publicas en una empresa

> [!info] Una red empresarial tiene dos zonas
> ```
> 🔒 INTRANET (red interna)     → direcciones PRIVADAS
> 🌐 DMZ (servidores publicos)  → direcciones PUBLICAS
> ```

> [!example] Estructura tipica
> ```
> Empresa usa 10.0.0.0/8 internamente
> → subred en /16 o /24 segun necesidad
>
> Servidores DMZ (web, correo publico)
> → direcciones IP publicas (accesibles desde Internet)
> ```

---

## ⚖️ Los 2 factores al planificar subredes

> [!warning] Siempre hay un trade-off
> Mas bits para subredes = menos hosts por subred, y viceversa.
> ```
> 1. ¿Cuantos HOSTS necesita cada red?
> 2. ¿Cuantas SUBREDES individuales necesito?
> ```

---

## 🎯 Ejemplo completo de subneteo eficiente

> [!example] ISP asigna 172.16.0.0/22 a una sede
> ```
> Direccion asignada: 172.16.0.0/22 (1,022 hosts disponibles)
>
> Requisitos:
> • 5 sitios = 5 conexiones a Internet
> • Necesito 10 subredes
> • La subred mas grande necesita 40 direcciones
>
> Solucion: mascara /26 (255.255.255.192)
> • /26 da 62 hosts utilizables → cubre los 40 ✅
> • Genera suficientes subredes para los 10 ✅
> ```

---

## 🧠 Metodologia paso a paso

> [!tip] 💡 Como abordar cualquier problema de subneteo
> ```
> PASO 1 → Cuenta cuantas subredes necesitas
> PASO 2 → Cuenta cuantos hosts necesita la subred MAS GRANDE
> PASO 3 → Elige la mascara que cumpla AMBOS requisitos
> PASO 4 → Usa el numero magico para listar las subredes
> PASO 5 → Asigna cada subred a su segmento de red
> ```
